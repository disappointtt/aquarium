# Умный аквариум

Flutter-приложение для мониторинга и управления аквариумом через ESP-контроллер. Приложение показывает текущие параметры воды, управляет оборудованием, ведет историю событий и предупреждает о критических состояниях.

## Возможности

- получение данных с ESP через HTTP;
- отображение температуры, уровня воды, состояния света, компрессора, помпы и режима системы;
- управление светом, компрессором и направлением потока;
- быстрые действия: обновление данных, повторное подключение, синхронизация времени, аварийное выключение;
- пресеты работы аквариума: Day, Night, Feeding;
- аналитика по изменению температуры и уровня воды;
- история команд, показаний и предупреждений;
- экспорт истории в JSON или CSV через буфер обмена;
- настройка IP-адреса ESP и проверка подключения;
- демо-режим с имитацией данных без ESP;
- светлая, темная и системная тема.

## Уведомления

Приложение создает предупреждения, показывает их на главном экране, выводит `SnackBar` и записывает событие в историю.

Условия предупреждений:

- уровень воды ниже `80%`;
- уровень воды выше `95%`;
- температура ниже `15 C`;
- температура выше `20 C`;
- свет выключен дольше `3 минут`.

Если проблема остается активной, одно и то же предупреждение повторяется не чаще одного раза в `2 минуты`. Например, сообщение `Вода слишком горячая` будет приходить повторно каждые 2 минуты, пока температура остается выше `20 C`.

## Экраны

### Home

Главный экран с текущим состоянием аквариума:

- карточки температуры и уровня воды;
- статус подключения и причина ошибки при offline-состоянии;
- активное предупреждение;
- статус системы;
- аналитика;
- управление светом, компрессором и потоком;
- пресеты и быстрые команды;
- последние события истории.

### Aquarium

Экран-заготовка для будущего выбора нескольких аквариумов. Сейчас показывает одну карточку аквариума.

### History

История событий приложения:

- фильтры `All`, `Commands`, `Alerts`, `Reads`;
- сортировка `DESC` и `ASC`;
- группировка по дням;
- экспорт событий в JSON или CSV.

### Settings

Настройки приложения:

- IP-адрес ESP;
- кнопка проверки подключения к `/status`;
- выбор темы;
- включение демо-режима;
- информация о версии.

## HTTP API ESP

Приложение ожидает, что ESP доступен по адресу:

```text
http://<ESP_IP>
```

Используемые endpoints:

```text
GET /status
GET /analytics
GET /led?state=on|off
GET /motor?state=on|off&dir=forward|back
GET /compressor?state=on|off
```

Основные поля ответа `/status`, которые используются в приложении:

```json
{
  "led": true,
  "temp": 19.5,
  "levelPercent": 90,
  "pump": false,
  "compressor": true,
  "mode": "IDLE",
  "time": "12:00:00",
  "timeSynced": true,
  "historyCount": 10,
  "cleanProgress": 0,
  "canClean": true,
  "motor": 0
}
```

Поля ответа `/analytics`:

```json
{
  "tempSlope": 0.0,
  "levelSlope": 0.0,
  "daysToLowTemp": null,
  "daysToLowLevel": null
}
```

## Технологии

- Flutter / Dart;
- `http` для запросов к ESP;
- `shared_preferences` для сохранения IP, темы и демо-режима;
- Material Design компоненты Flutter.

## Запуск проекта

Установить зависимости:

```bash
flutter pub get
```

Запустить приложение:

```bash
flutter run
```

Запустить тесты:

```bash
flutter test
```

Проверить анализатор:

```bash
flutter analyze
```

## Структура проекта

```text
lib/
  main.dart                  # состояние приложения, тема, SharedPreferences
  data/history_store.dart    # in-memory история событий
  models/history_models.dart # модели истории и снимков состояния
  screens/
    app_shell.dart           # нижняя навигация
    home_screen.dart         # мониторинг, команды и уведомления
    aquariums_screen.dart    # экран аквариума
    history_screen.dart      # история и экспорт
    settings_screen.dart     # настройки
  widgets/ui_components.dart # общие UI-компоненты
```

## Примечания

- История хранится в памяти приложения и ограничена последними 200 событиями.
- Демо-режим позволяет проверять интерфейс без подключенного ESP.
- Для реального режима нужно указать IP-адрес ESP на экране Settings.
